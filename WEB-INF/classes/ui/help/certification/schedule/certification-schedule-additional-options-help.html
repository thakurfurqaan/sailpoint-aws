Additional Settings

Certification Name: A name and date parameter to identify the certification.

Certification Owner: The identity or workgroup responsible for the certification.

Advanced Options

  • Enable Bulk Clear Decisions: Allow certifiers to cancel multiple decisions simultaneously in the access review.
  • Update Entitlement Assignments: Enable this to cause decisions made on entitlement values in the access review to apply to the entitlement assignment model. When this is enabled, approvals create assignments, and revocations remove assignments.
  • Enable Partitioning: Partitioning aids the performance of certification scheduling, by subdividing activity across multiple threads, to increase processing throughput and speed. If you enable partitioning, you also set the Number of Partitions.
  • Show Recommendations: Enable recommendations from IdentityAI to appear in access reviews. This option is visible only if you have implemented SailPoint IdentityAI.
    • Automatically Approve Recommended Items: Automatically mark access review items that are recommended for approval as Approved, and move them from the Open to the Review tab of the access review. This option is visible only if you have implemented SailPoint IdentityAI.
  • Show Classifications: Show classification information in the access reviews. When enabled, classifications provide additional information about roles, managed attributes and policy violations.
  • Delegation Options
  • Delegating an item to another user allows that user to make the decisions and return the review to the original reviewer, who can then accept or change the delegate reviewer’s decisions, and sign off. With delegation, the original reviewer still retains ultimate control of the decision and the sign off.
    • Require Delegation Review: Require the original access review owner to review all delegated access reviews.
    • Line Item Delegation: Allow certifiers to delegate individual items from an access review.
    • Identity Delegation: Allow certifiers to delegate entire identities from an access review.
    • Disable Delegation Forwarding: Disallows the forwarding of a work item that was delegated by a different user.
    • Pre-Delegation Rule: A rule to pre-delegate entities to specific reviewers. Rules of type “CertificationPreDelegation” are included in this list. In the Targeted Certification, the pre-delegation rule does not support reassignments; for reassignments, use the Certifier Rule in the Choose Certifier section.
    • Email Owner on Pre-Delegation Completion: If you are using a Pre-Delegation Rule, this option sends an email to the owner of the original certification when delegate reviewers have completed their certifications.
  • Approve Options
    • Require Comments with Approval: Require the certifier to include comments when an access review item is approved.
    • Enable Bulk Approval: Allow certifiers to approve multiple decisions simultaneously in the access review.
  • Revoke Options
    • Enable Bulk Revocation: Allow certifiers to revoke multiple items simultaneously in the access review.
    • Enable Account Revocation: Allow certifiers to revoke an account and all its associated entitlements.
    • Enable Bulk Account Revocation: Allow certifiers to revoke multiple accounts simultaneously in the access review.
    • Require Comments with Revocation: Require the certifier to include comments when an access review item is revoked.
  • Allow Options
    • Enable Allow Exceptions: Allow exceptions on non-policy-violation items such as roles or entitlements. Allowing an exception means the user should not have access indefinitely, but can retain access for a specified period of time. Exceptions for policy violations are always enabled, so note that this setting applies only to items that are not policy violations.
    • Enable Bulk Allow Exceptions: Allow certifiers to approve multiple exceptions simultaneously in the access review.
    • Deprovision Items When Exception Expires: Enables automatic deprovisioning of access when the allowed exception period has expired. This setting applies only to items such as roles or entitlements, that are not policy violations.
    • Enable Allow Exception Popup: The Allow Exception popup lets reviewers manually set expiration dates and allow comments. This option applies to both violation and non-violation items.
    • Require Comments for Allowing Exceptions: Require the certifier to include comments when an exception is allowed. . This option applies to both violation and non-violation items.
    • Default Duration for Exceptions: The default time period in which exceptions are allowed during the access review. The reviewer can modify this default time period during the review, in the Allow Exceptions pop-up. After this time period, exceptions will expire.
  • Access Review Properties
    • Custom Name: A custom name for the access reviews generated by this certification. You can combine free text, and parameterized text selected from the list. The Custom Name appears in the reviewer’s My Access Reviews page.
    • Custom Short Name: A custom short name for the access reviews generated by this certification. You can combine free text, and parameterized text selected from the list. The Custom Short Name appears in the user’s detailed Access Review page.
  • Tags: Labels that are used to classify certifications for searching and reporting.
